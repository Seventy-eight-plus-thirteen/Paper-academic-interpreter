# 思维导图生成规则与Prompt工程规范

## 一、思维导图结构设计原则

### 1.1 核心设计原则

#### 1.1.1 节点命名规范
- **禁止使用**：要点1、要点2、内容1、内容2等无意义占位符
- **必须使用**：具体、准确、有信息量的专业术语或概念
- **命名标准**：
  - 医学文献：使用疾病名称、诊断方法、治疗策略等专业术语
  - 技术论文：使用技术概念、算法名称、架构组件等
  - 综述文章：使用主题分类、关键发现、发展趋势等

#### 1.1.2 层次结构规范
```
根节点（论文标题）
├── 阶段1：预读/概述（3-4个子节点）
│   ├── 子主题1（具体概念）
│   │   ├── 要点A（细节描述）
│   │   └── 要点B（细节描述）
│   └── 子主题2（具体概念）
├── 阶段2：结构/方法（3-4个子节点）
├── 阶段3：深度/细节（3-4个子节点）
├── 阶段4：批判/分析（3-4个子节点）
└── 阶段5：延伸/展望（3-4个子节点）
```

#### 1.1.3 内容质量标准
- **描述字段**：必须包含具体内容，不是泛泛而谈
- **笔记字段**：摘录原文关键信息，保持准确性
- **专业术语**：使用领域内的标准术语

MINDMAP_GENERATION_PROMPT = """

## 1. 结构规范
必须严格遵循Pretext框架的5个阶段:
1. 📖 预读阶段 - 快速了解基本信息和核心观点
2. 🏗️ 结构分析 - 整体框架和各部分主要内容
3. 🔍 深度阅读 - 核心概念和技术实现细节
4. 💡 批判思考 - 创新点、局限性和对比分析
5. 🚀 延伸思考 - 应用场景、未来方向和领域影响

## 2. 节点命名规则（极其重要）

### 禁止使用的节点名称:
- ❌ 要点1、要点2、要点3...
- ❌ 内容1、内容2、内容3...
- ❌ 信息A、信息B、信息C...
- ❌ 发现1、发现2、发现3...
- ❌ 任何数字编号或字母编号的占位符
- ❌ 临床意义、诊断要点（作为叶子节点名称时禁止，必须展开为具体内容）

### 必须使用的节点名称:
- ✅ 具体的概念名称（如"自注意力机制"、"TNM分期"）
- ✅ 专业术语（如"残差连接"、"EGFR突变"）
- ✅ 方法/技术名称（如"Transformer架构"、"免疫组化"）
- ✅ 分类/类别名称（如"上皮性肿瘤"、"神经内分泌肿瘤"）
- ✅ 从原文中提取的关键主题词

## 3. 内容深度要求（极其重要）

### 医学领域内容深度标准:
每个诊断/治疗/检查相关节点必须包含以下维度的详细内容：

#### 对于"临床表现"类节点：
- description: 必须列举具体症状（如"咳嗽、咯血、胸痛、呼吸困难"）
- note: 必须包含：发病机制、典型体征、严重程度分级、与其他疾病的鉴别要点

#### 对于"影像学检查"类节点：
- description: 必须说明检查目的和适用场景
- note: 必须包含：检查方法细节、典型影像表现、诊断标准、敏感性和特异性数据

#### 对于"病理诊断"类节点：
- description: 必须说明诊断方法和标本类型
- note: 必须包含：病理特征描述、分级标准、免疫组化标志物、分子病理特征

#### 对于"治疗策略"类节点：
- description: 必须说明治疗方案和适用人群
- note: 必须包含：具体药物/手术方式、剂量/疗程、疗效数据、副作用管理

### 禁止内容模板:
❌ "该要点的临床应用价值"
❌ "诊断时需要注意的关键点"
❌ "相关内容"
❌ "补充信息"
❌ "详细说明"

### 正确内容示例:
✅ "临床意义：该指标升高提示肿瘤负荷增加，可用于疗效监测和预后评估。正常值<10ng/mL，>100ng/mL提示广泛转移"
✅ "诊断要点：①年龄>40岁吸烟者；②持续性咳嗽>2周；③CT显示肺门肿块伴纵隔淋巴结肿大；④需与肺结核、肺炎鉴别"

## 4. 内容提取规则

### 从原文提取节点名称的方法:
1. 识别章节标题中的核心概念
2. 提取关键段落的第一句主题句
3. 找出重复出现的重要术语
4. 识别方法、技术、分类的具体名称
5. 使用专业领域内的标准命名

### 描述内容生成规则:
- 描述必须是具体的，不是泛泛的
- 从原文摘录关键信息
- 使用准确的数字、数据、术语
- 保持医学/技术准确性
- 每个note字段至少50-100字的具体内容

```python
def post_process_mindmap(structure: Dict) -> Dict:
    """后处理思维导图结构"""
    
    def process_node(node: Dict, path: str = ""):
        """递归处理节点"""
        name = node.get('name', '')
        
        # 检查并修复占位符名称
        if is_placeholder_name(name):
            # 从description或note中提取更好的名称
            better_name = extract_better_name(node)
            node['name'] = better_name
        
        # 处理子节点
        for child in node.get('children', []):
            process_node(child, f"{path}/{name}")
    
    process_node(structure)
    return structure

def is_placeholder_name(name: str) -> bool:
    """检查是否是占位符名称"""
    placeholder_patterns = [
        r'要点\d+',
        r'内容\d+',
        r'信息\w*\d*',
        r'发现\d+',
        r'关键\d+',
    ]
    for pattern in placeholder_patterns:
        if re.search(pattern, name):
            return True
    return False
```

## 四、评估与迭代

### 4.1 质量评估指标

1. **节点命名质量**
   - 占位符比例 < 5%
   - 专业术语使用率 > 80%
   - 平均节点名称长度 4-15字

2. **内容准确性**
   - 与原文一致性 > 90%
   - 描述具体性评分 > 4/5
   - 术语准确性 > 95%

3. **结构完整性**
   - 5个阶段覆盖率 100%
   - 平均每阶段子节点 3-5个
   - 层次深度 3-4层

### 4.2 迭代优化流程

```
1. 生成思维导图
2. 自动质量检查
3. 发现问题（占位符、空洞描述等）
4. 提取原文关键信息
5. 重新生成问题节点
6. 人工审核
7. 收集反馈
8. 优化Prompt
9. 重复步骤1
```

## 五、总结

### 核心原则
1. **具体优于抽象** - 使用具体概念，避免泛泛而谈
2. **专业术语优先** - 使用领域标准术语
3. **原文驱动** - 所有内容来自原文，不编造
4. **结构清晰** - 5阶段框架，层次明确
5. **质量第一** - 宁缺毋滥，确保每个节点都有价值

### 成功标准
- 用户看到思维导图能立即理解论文核心内容
- 每个节点名称都能准确传达信息
- 没有"要点1、2、3"等无意义占位符
- 专业术语使用准确，内容可信
